## 3. NEXT — 순서대로 (1순위: 출시)
1. **커밋**: 위 §2 파일들 리뷰 → `git add` → 커밋. (사람 확인 후. 변경요약 먼저 보고.)
2. **아이콘 .import 생성**: Godot 에디터 한 번 열어 `android_icons/*.png` 임포트되게.
3. **서명된 AAB 빌드** (로컬, 환경 이미 됨 — APK가 증거):
   ```bash
   cd godot
   export GODOT_ANDROID_KEYSTORE_RELEASE_PATH="$HOME/keystores/new_upload_key.jks"
   export GODOT_ANDROID_KEYSTORE_RELEASE_USER="<alias: keytool -list -v -keystore ...>"
   export GODOT_ANDROID_KEYSTORE_RELEASE_PASSWORD="••••"
   godot --headless --export-release "Android" ../build/android/hitezero.aab
   ```
   → 확인: APK 아닌 **.aab**, 그리고 **디버그 아닌 업로드키 서명**인지. (현 APK는 디버그 서명.)
   - 가이드: `godot/docs/google_play_release.md`(상세) / CI 대안: `godot/docs/android_ci.md`.
4. **Play Console 제출**: `godot/docs/play_console_submission.md` 답안 그대로.
   privacy URL = https://hitezero.netlify.app/privacy. **내부 테스트 트랙부터** → 실기 1회 풀플레이 → 프로덕션.

## 5. 지켜야 할 룰 (하드)
- **검증 없이 완료 선언 금지**: 빌드/익스포트 후 산출물·서명 실제 확인.
- **시크릿 커밋 금지**: `*.jks`/비번을 git·cfg에 남기지 마(.gitignore 이미 막음). CI는 GitHub Secrets.
- **게임플레이 계약 유지**: `godot/docs/gameplay_spec.md`의 로직 상수·상태머신 불변(비주얼 작업이 깨면 안 됨).
- **git 커밋은 사람 확인 후**: 변경 파일 목록 + 요약 보고 먼저.
- **Play엔 AAB만**(APK는 테스트용). versionCode는 업데이트마다 +1.
- 마운트/동기화 주의: `dist/godot-web`이 세션 중 사라진 적 있음 — 빌드 소스는 실재 확인 후 사용.
