# HiteZero — v1.1 광고 구현 가이드 (착수본)

> 짝 문서: [`v1.1_ads_plan.md`](v1.1_ads_plan.md) (스코프·포맷 결정). 이 문서는 *코드가 이미 들어간 상태*에서 v1.1에 켜는 절차다.
> 포맷: 보상형 "광고 보고 1회 부활"(rewarded). 배너 없음. SDK: godot-admob (Android 전용).

## 지금 들어가 있는 것 (v1.0 클린 유지)

부활 루프 코드는 **이미 배선됐고 기본 OFF**다. v1.0 빌드는 동작·컴파일·제출 프로필 전부 무변화.

| 파일 | 역할 | 상태 |
|---|---|---|
| `scripts/ads_manager.gd` | 오토로드 파사드. `ADS_ENABLED` 마스터 스위치. **애드온 심볼 0개** → 애드온 없이도 컴파일됨 | 라이브(OFF) |
| `scripts/game_root.gd` | `_trigger_game_over()`에 부활 오퍼 분기. 보상→`_resume_after_revive()`, 거절/실패→`_commit_game_over()`(원래 게임오버) | 라이브(게이트) |
| `scripts/revive_prompt.gd` | 최소 "광고 보고 부활?" 오퍼 UI(카운트다운+버튼). 비주얼 폴리시는 juice-smith 핸드오프 | 라이브(미발동) |
| `scripts/ads_backend.gd.txt` | AdMob 노드/SDK를 만지는 래퍼. **`.txt`라 컴파일 안 됨** → v1.1에서 `.gd`로 활성화 | 템플릿 |
| `project.godot` | `AdsManager` 오토로드 등록 | 라이브 |
| `export_presets.cfg` | **손 안 댐** — INTERNET 권한 없음·code 1·1.0.0 그대로 | 무변화 |

게이트 로직: `_trigger_game_over()`는 `AdsManager.is_revive_available()`가 true이고 이번 런에서 부활을 안 썼을 때만 오퍼한다. `ADS_ENABLED=false`면 `is_revive_available()`가 항상 false라 원래 게임오버 경로로 직행한다.

## v1.1 활성화 5단계

### 1) 애드온 설치 + Gradle 빌드 활성
- [godot-admob](https://github.com/godot-sdk-integrations/godot-admob) 설치 ([설치 문서](https://godot-sdk-integrations.github.io/godot-admob/getting-started/installation.html)). AssetLib 또는 릴리스 zip → `res://addons/admob/`.
- Project → Install Android Build Template, Export → Android 프리셋에서 **Gradle Build = On** (AdMob은 커스텀 Android 빌드 필수).
- AdMob 계정에서 앱 등록 → **App ID**와 **Rewarded 광고단위 ID** 발급.

### 2) Admob 노드 추가 + 그룹 지정
- 메인 씬(예: `scenes/boot.tscn` 루트 또는 항상 살아있는 노드)에 **`Admob` 노드** 추가.
- 그 노드를 그룹 **`admob`** 에 넣는다 (백엔드가 `get_first_node_in_group("admob")`로 찾음).
- 인스펙터에서 Rewarded 광고단위 ID 채우기:
  - **Debug ID**(디버그 빌드에서만 사용) = 구글 공식 테스트 ID로 시작 → 개발 중엔 항상 테스트 광고가 떠서 계정 정지 위험 0:
    - App ID: `ca-app-pub-3940256099942544~3347511713`
    - Rewarded: `ca-app-pub-3940256099942544/5224354917`
  - **Real ID** = 실제 발급받은 광고단위. `is_real` 필드는 **출시 직전에만** true로.

### 3) 백엔드 템플릿 활성화
```
scripts/ads_backend.gd.txt  →  scripts/ads_backend.gd   (리네임만)
```
리네임하면 AdMob API를 만지는 코드가 컴파일된다. `ads_backend.gd` 안의 `ad_info.ad_id` 필드명은 설치한 애드온 버전의 `AdInfo` 클래스와 한 번 대조(문서 [Classes](https://godot-sdk-integrations.github.io/godot-admob/api/classes.html)) — 시그널/메서드명은 검증됨, id 필드만 확인.

### 4) 마스터 스위치 ON
`scripts/ads_manager.gd`:
```gdscript
const ADS_ENABLED := true   # false → true
```

### 5) 제출 프로필 6변경 (출시 전 필수 — 클린→광고 전환)
`v1.1_ads_plan.md`와 동일. 광고 켜면 v1.0의 "완전 오프라인/무수집/무광고" 프로필이 깨지므로 전부 처리:

1. **INTERNET 권한 추가** — `export_presets.cfg`: `permissions/internet=true` (또는 custom_permissions에 `android.permission.INTERNET`). *이 변경은 활성화 전까지 하지 말 것.*
2. **데이터 안전 폼 재작성** — "수집 안 함" → "광고 ID·기기정보 수집/공유"(AdMob 기준).
3. **광고 포함 선언**: 아니요 → **예**.
4. **개인정보처리방침 재작성** — AdMob SDK 데이터 수집·제3자 공유 조항 (`store_assets/privacy.html`).
5. **콘텐츠 등급(IARC) 재설문** — 광고 포함 항목 갱신.
6. **스토어 카피 수정** — "광고 없음" 문구 폐기(상세/간단설명 EN·KR). `versionCode +1`, `versionName 1.1.0`.

> Steam 빌드: AdMob은 Android 전용이라 데스크톱/웹 빌드엔 안 붙는다. `is_revive_available()`가 그 플랫폼에선 자연히 false. 별도 분기 불필요(애드온 노드 미존재 → 백엔드 disabled).

## 실기기 테스트 체크리스트 (tech-lead 핸드오프)

샌드박스엔 Godot 바이너리가 없어 아래는 **에디터+실기기에서 확인 필요**:

- [ ] `ADS_ENABLED=false` 상태로 에디터 실행 → 게임오버가 기존과 100% 동일(오퍼 안 뜸). *무변화 회귀 확인.*
- [ ] `ADS_ENABLED=true` + 테스트 ID → 게임오버 시 "광고 보고 부활?" 오퍼 표시.
- [ ] 부활 수락 → 테스트 보상광고 재생 → 시청 완료 시 하트 1 복구·`AIMING` 복귀, 점수/스테이지 유지.
- [ ] 광고 중간 닫기(보상 미획득) → 정상 게임오버로 폴백.
- [ ] 부활은 **런당 1회**만(두 번째 게임오버엔 오퍼 안 뜸).
- [ ] 광고 로드 실패 시 오퍼 안 뜨고 정상 게임오버(앱 멈춤 없음).
- [ ] 세이브 호환: `user://save.cfg`(ConfigFile) 로드/세이브 정상 — 광고는 세이브 포맷 무관.
- [ ] 실기기 1회 풀플레이(부활 포함) + 스토어 제출 가능 빌드 산출.
