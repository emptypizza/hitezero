# HiteZero — v1.1 광고 계획 (PARKED)

> 결정 2026-06-15. **v1.0은 무광고 클린 출시**(제출 준비 끝남). 광고는 v1.1로 연기.
> 트리거: 출시 후 **DAU가 의미 있게 쌓인 뒤** 착수. 설치 0 단계에선 어떤 광고도 수익 ≈ 0이라 통합 가치 없음.

## 포맷 결정
- **보상형(rewarded) "광고 보고 1회 부활" = 1순위.** 옵트인이라 유저 반발 없음, eCPM 최고, 하트3 점수도전 루프(죽음→아쉬움)에 정확히 붙음.
- **전면(interstitial) = 선택.** 넣으면 빈도 제한 필수(게임오버 3회당 1회·최소 60~90초 간격). 무제한은 평점·리텐션 즉사.
- **하단 배너 = 안 함.** 하단이 액티브 영역(메이드·트레이)이라 플레이 가림 + eCPM 최저.

## 광고 붙이면 같이 바꿔야 하는 것 (제출 프로필 전면 변경 — 6개)
1. **INTERNET 권한 추가** (현재 완전 오프라인·권한 없음).
2. **데이터 안전 폼 재작성**: "수집 안 함" → "광고 ID·기기정보 수집/공유"(AdMob 기준).
3. **광고 선언**: 아니요 → **예**.
4. **개인정보처리방침 재작성**: AdMob SDK 데이터 수집·제3자 공유 조항 추가 (`store_assets/privacy.html`).
5. **콘텐츠 등급(IARC) 재설문**: 광고 포함 항목 갱신.
6. **스토어 카피 수정**: "광고 없음" 문구 폐기(상세설명·간단설명 EN/KR 양쪽).

## 통합 터치포인트 (구현 시)
- SDK: AdMob (GPGS v2 동거). Steam 빌드엔 광고 비활성 분기 유지.
- 코드: 게임오버 → 부활 제안 UI → rewarded 콜백 성공 시 하트 1 복구·이어하기. 실패/미시청 시 정상 게임오버.
- versionCode +1 (=2), versionName 1.1.0.
- 게임플레이 계약: 부활은 신규 룰 → `gameplay_spec.md` 갱신.

## 하지 말 것
- v1.0 빌드에 광고 코드/권한/SDK **추가 금지**. 클린 제출이 깨진다.
