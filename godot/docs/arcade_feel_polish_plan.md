# HiteZero — 아케이드 손맛/연출 폴리싱 개선안

> **목표:** 게임 규칙은 거의 건드리지 않고, 타격감(impact)·연출(juice)·피드백 레이어만 폴리싱해서 "한 번 치면 손맛이 온다"를 만든다.
> **범위:** Godot 4.6 / GDScript, GL Compatibility, 웹(no-threads) + 모바일. 결정론적 게임플레이 유지.
> **작성 기준:** 2026-05-29 시점 `godot/scripts/` 실제 코드 분석.

---

## 1. 현황 진단

이미 손맛 베이스는 깔려 있다. 빈 캔버스가 아니라 **튜닝/재설계 단계**다.

### 이미 구현된 것 (유지·강화 대상)
- **스크린 셰이크** — `game_root.gd:_kick_world()` / `_update_effects()`. 방향 벡터 + per-frame 지터를 `world.position`에 적용, 선형 감쇠(`t`).
- **임팩트 버스트** — `_burst_feedback()` / `_draw_impact_bursts()`. 확장하는 원형 플래시.
- **나이프 트레일** — `knife.gd:_draw_trail()`. 글로우/아웃라인/코어 3중 라인, 8포인트 히스토리, 미니 나이프는 더 가볍게.
- **플레이어 발사 VFX** — `player.gd:_draw_output_vfx()`. 머즐 플래시 + 슬래시 라인 + 스파클 파티클 18개 + 리코일 오프셋.
- **블록 히트 피드백** — `block.gd`. `flash_amount` 색 플래시 + 사인 기반 스케일 펄스.
- **히트스탑 / 프리즈 프레임** — `_freeze_frame()` (전역 `Engine.time_scale=0.05`), `_flash_screen()` (t² 이징, 실시간 msec).
- **콤보 시스템** — `game_constants.gd`에 윈도우(0.8s)·티어(3/6/10/15)·배율(1.0~5.0)·색상 정의, `_register_combo_hit()` / `_spawn_combo_text()`.
- **즉시 발사 큐** — `_play_release_commit_cue()`. 첫 나이프 스폰(66ms) 전에 0.1s 인지 한계 안에서 피드백 선발.
- **절차적 SFX** — `audio_manager.gd`. 8종(launch/hit/destroy×3/clear/over/bounce) 고정 클립·고정 볼륨.

### 부족한 것 (이번 폴리싱의 핵심 갭)
1. **일반 블록 파괴에 히트스탑이 없다.** 마이크로 프리즈는 POW(0.05s)·보스·게임오버에만. 아케이드 "탁!" 손맛의 핵심인 30~60ms 임팩트 프리즈가 일반 타격엔 빠져 있음.
2. **히트스탑이 전역·비중첩.** `_freeze_frame()`은 `Engine.time_scale`을 전역으로 0.05로 떨어뜨려 HUD 트윈·콤보 텍스트까지 같이 멈춤. 게다가 `if Engine.time_scale < 0.5: return` 가드 때문에 **연속 타격 시 프리즈가 스택되지 않고 버려진다** → 콤보 구간에서 손맛이 평탄해짐.
3. **셰이크가 위치 전용·선형 감쇠.** 회전/줌 킥 없음, per-frame uniform 지터라 "버징"하게 보임. trauma² 모델이 더 부드럽고 강도 스택이 자연스러움.
4. **블록 반응이 약하다.** 맞을 때 색 플래시 + 사인 펄스뿐. 임팩트 방향으로의 스쿼시&스트레치(젤리 눌림)·넉백·방향성 스파크가 없어 "찌르는" 촉감이 부족.
5. **SFX에 다이내믹스가 없다.** 매번 동일 피치·동일 볼륨. 콤보 피치 램프(연타마다 음정 상승)·임팩트 서브 레이어가 없음 — 저공수 고효율 손맛 누락.
6. **모바일 햅틱 없음.** 웹/모바일 타겟인데 진동 피드백(`Input.vibrate_handheld` / JS `navigator.vibrate`)이 전무.
7. **2D 셰이더/라이팅 미사용.** 네온 프레임·그리드가 손그림 라인 + TextureRect. 글로우/블룸 셰이더, Light2D 임팩트 라이팅 없음. (사용자 성장 목표: 2D Shader·lighting)
8. **튠 값이 1666줄에 흩어진 매직넘버.** 셰이크 강도 2/3/4, 프리즈 0.05/0.15/0.2/0.4, 플래시 강도가 함수마다 하드코딩 → 손맛 이터레이션 때마다 코드를 뒤져야 함.

## 2. 개선 방향 요약

핵심 원칙: **"매 타격에 짧고 강한 멀티 채널 피드백(비주얼+오디오+촉각)을 주되, 강도는 임팩트에 비례하고 콤보로 누적된다."**

- **타격감 3단 강화**: 마이크로 히트스탑(스택형) → 블록 스쿼시&넉백 → 방향성 스파크.
- **피드백 다이내믹스**: 콤보가 오를수록 셰이크·피치·플래시·시간왜곡이 함께 고조.
- **연출 레이어**: 네온 글로우/블룸 셰이더 + 임팩트 라이트로 "주스" 한 단계 업.
- **모바일 손맛**: 햅틱 추가.
- **이터레이션 속도**: 모든 튠 값을 단일 `GameFeel` 리소스로 통합.

### 업무 1 — 마이크로 히트스탑 재설계 (스택형, 게임플레이 한정)

**문제.** `_freeze_frame()`(`game_root.gd:1447`)이 전역 `Engine.time_scale=0.05`를 쓰고 `if Engine.time_scale < 0.5: return` 가드로 중첩을 버린다. 일반 블록 파괴(`_destroy_block()`)엔 프리즈가 아예 없다.

**할 일.**
- 전역 time_scale 대신 게임플레이 레이어(`world`)에만 적용되는 hit-stop 누산기 도입(프레임 카운트 또는 실시간 msec 기반, HUD/입력은 살아 있게).
- 가드를 "스택"으로 변경 — 남은 프리즈 시간 = `max(remaining, new)`가 아니라 **임팩트별 누적 + 상한**으로 연타 시 묵직함 유지.
- `_hit_block()` / `_destroy_block()`에 임팩트 비례 마이크로 프리즈(일반 30ms, 파괴 50ms, POW/보스는 기존 유지) 추가.
- 콤보 티어가 오를수록 프리즈 길이 소폭 가산.

**수용 기준.** 일반 블록 타격·파괴 시 체감되는 미세 정지가 발생하고, HUD 콤보 텍스트/타이머는 멈추지 않는다. 연타 시 프리즈가 끊기지 않는다(가드로 드롭 0건).

---

### 업무 2 — SFX 다이내믹스: 콤보 피치 램프 + 임팩트 레이어

**문제.** `audio_manager.gd`는 `_add(name, stream, volume)`로 고정 피치·고정 볼륨 재생. 연타에도 소리가 똑같아 손맛이 평탄.

**할 일.**
- `AudioManager.play(name, pitch=1.0, volume_db_offset=0.0)`로 시그니처 확장(`AudioStreamPlayer.pitch_scale` 활용).
- `block_hit` / `block_destroy_*`를 현재 콤보 카운트에 비례해 피치 업(예: `1.0 + min(combo, 12) * 0.03`), 콤보 끊기면 리셋.
- 파괴음에 저주파 "서브 텀프" 레이어 1개 추가(`_gen_*`에 짧은 사인 바디).
- 동시 다중 히트 대비 풀(`AudioStreamPlayer` 폴리포니) 확인.

**수용 기준.** 연타 시 타격음 음정이 단계적으로 상승하고, 콤보 종료 시 리셋. 파괴음에 무게감(저역) 추가. 동시 5타에도 클리핑/드롭 없음.

---

### 업무 3 — 블록 스쿼시&스트레치 + 방향성 임팩트 스파크

**문제.** `block.gd`는 맞을 때 `flash_amount` 색 플래시 + 사인 펄스(`_update_sprite_scale()`)만. 찌르는 촉감이 없다.

**할 일.**
- `block.gd`에 `take_hit()` 시 임팩트 방향 기반 스쿼시&스트레치(피격 축으로 눌렸다 복원, 0.12s 트윈) + 미세 넉백 오프셋(복원) 추가.
- `game_root.gd:_hit_block()`에서 나이프 속도 방향을 블록에 전달(현재 `block.global_position`만 넘김).
- `_burst_feedback`을 방향성 스파크로 보강 — 충돌 노멀 반대 방향으로 짧은 스파크 라인 2~3개(`player.gd:_draw_star` 패턴 재사용).

**수용 기준.** 블록이 맞은 방향으로 눌렸다 돌아오고, 스파크가 충돌 방향성을 가진다. 파괴되지 않는 타격에서도 명확히 "맞았다"가 읽힌다.

---

### 업무 4 — 스크린 셰이크 trauma² 모델 + 회전/줌 킥

**문제.** `_update_effects()`의 셰이크는 선형 감쇠 + uniform per-frame 지터 + 위치 전용. 강도 누적/감쇠가 거칠고 "버징"함.

**할 일.**
- `shake_strength/duration` 누산을 **trauma(0~1) 모델**로 교체: 표시 강도 = `trauma²`, 매 프레임 `trauma -= decay*delta`.
- per-frame `randf`를 시간 시드 노이즈(`FastNoiseLite` 또는 누적 위상)로 교체 → 부드러운 흔들림.
- 큰 임팩트(파괴·POW·보스)에 미세 회전 킥(`world.rotation`, ±1.5° 이내)·줌 킥(`world.scale` 1.0→1.02) 옵션 추가, 짧게 복원.
- `_kick_world()` 호출부는 trauma 추가량으로 매핑(기존 strength 2/3/4 → trauma 0.15/0.25/0.4).

**수용 기준.** 동일 강도에서 셰이크가 더 매끄럽고, 연속 타격 시 trauma가 자연 누적·감쇠. 회전/줌 킥이 과하지 않게(멀미 방지) 클램프됨.

---

### 업무 5 — 모바일 햅틱 피드백

**문제.** 웹/모바일 타겟인데 촉각 피드백 0. 손맛의 한 축이 비어 있음.

**할 일.**
- 신규 `scripts/haptics.gd`(autoload) — 네이티브는 `Input.vibrate_handheld(ms)`, 웹은 `JavaScriptBridge.eval("navigator.vibrate(...)")`로 분기, 미지원 시 no-op.
- 강도 프리셋: light(타격), medium(파괴/트레이 바운스), heavy(하트 손실/보스 처치).
- `game_root.gd`의 `_hit_block`/`_destroy_block`/`_play_hit_reaction`/보스 처치에 훅 연결.
- 설정에서 on/off 토글(과민 사용자 대비), 기본 on.

**수용 기준.** 모바일 브라우저(Android Chrome)에서 타격/파괴/피격 시 진동 발생, 데스크탑/미지원 환경에서 에러 없이 무시. 토글 동작.

---

### 업무 6 — 2D 네온 글로우/블룸 셰이더 + Light2D 임팩트 라이팅

**문제.** 네온 컨셉인데 실제 글로우가 없다. 배경은 `_draw_background()`의 손그림 그리드 + TextureRect. (사용자 성장 목표: 2D Shader·lighting 직결)

**할 일.**
- CanvasItem `.gdshader`로 네온 프레임/스타·POW 블록에 가산(additive) 글로우/펄스 — GL Compatibility 호환 범위 내(텍스처 샘플 + 시간 기반 emissive 가산).
- 파괴·POW 순간 `PointLight2D`를 짧게 펄스시켜 주변을 물들이는 임팩트 라이팅(풀링으로 재사용).
- 웹 성능 가드: 셰이더/라이트는 품질 옵션으로 묶고, no-threads 웹에서 프레임 저하 시 자동 강등(저사양 폴백 = 기존 드로잉).

**수용 기준.** 네온 요소가 발광하고 임팩트 시 국소 라이트가 번쩍인다. 웹(no-threads)에서 60fps 유지, 저사양 폴백 동작. 콘솔 셰이더 에러 0.

---

### 업무 7 — `GameFeel` 튜닝 리소스 통합 + 손맛 회귀 검증

**문제.** 셰이크 강도·프리즈 길이·플래시 강도·트레일 폭 등 손맛 값이 `game_root.gd`(1666줄)·`knife.gd`·`block.gd`에 매직넘버로 흩어져 있어 이터레이션이 느림.

**할 일.**
- `Resource` 기반 `scripts/game_feel.gd`(또는 `.tres`)에 hit-stop/shake/flash/trail/haptic/combo-feel 파라미터를 한곳에 모으고 인스펙터에서 튜닝 가능하게.
- 업무 1~5의 신규 값들을 이 리소스에서 읽도록 리팩토링(기존 동작 보존).
- `godot/tools/`의 회귀 테스트 패턴(`test_six_axis_quality.py` 등) 확장 — 신규 손맛 훅(스택형 hit-stop, 햅틱 호출, 피치 램프, 스쿼시) 존재를 검사하는 테스트 추가.
- `--headless --check-only`로 전 스크립트 파싱 + 기존 테스트 그린 확인 후 웹 빌드 재생성.

**수용 기준.** 모든 손맛 값을 한 리소스에서 조정 가능. 기존 회귀 테스트 + 신규 손맛 테스트 그린. 웹 빌드 콘솔 에러 0, 게임플레이 규칙 동작 불변.

## 4. 통합 검증

- **자동:** `python3 godot/tools/test_six_axis_quality.py` 외 기존 테스트 + 신규 손맛 테스트 그린, `godot --headless --check-only` 전 스크립트 파싱 통과.
- **수동 QA:** AIMING→SHOOTING 드래그 손맛, 연타 콤보 구간의 hit-stop 스택·피치 램프 체감, 모바일 터치 + 햅틱, 웹 no-threads 60fps.
- **회귀 가드:** 게임플레이 규칙(`gameplay_spec.md`) 불변 — 충돌/점수/승패 타이밍은 건드리지 않는다.

## 5. 권장 진행 순서

저위험·고체감부터: **업무 1 → 2 → 3**(코어 타격감)으로 손맛 즉시 끌어올린 뒤, **4 → 5**(셰이크·햅틱)로 다채널 보강, **6**(셰이더/라이팅)으로 연출 레벨업, **7**(리소스 통합·검증)로 이터레이션 기반과 안전망을 마지막에 고정한다.
